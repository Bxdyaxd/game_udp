## 🎯 Описание

Это консольный клиент для игры в кости, где два игрока по очереди бросают кубик и размещают выпавшие значения на своем поле 3×3.

**Цель игры:** набрать максимальное количество очков, заполняя поле комбинациями.

## 💻 Системные требования

- **Операционная система:** Linux / macOS (с поддержкой termios)
- **Компилятор:** GCC или Clang
- **Библиотеки:** стандартные (stdio, stdlib, string, unistd, arpa/inet, termios)

📝 Команды
Команда	Описание
ENTER (пустая строка)	Бросить кубик (когда ваш ход)
rating	Показать таблицу рейтингов
help	Показать справку
quit / exit / q	Выйти из игры
📡 Протокол взаимодействия
Клиент общается с сервером через UDP на уровне команд:

От клиента к серверу:
text
LOGIN <username> <password>  - Авторизация
ROLL                         - Бросок кубика
PLACE <колонка> <значение>   - Размещение числа
RATING                       - Запрос рейтинга
От сервера к клиенту:
text
LOGIN_OK                     - Успешный вход
START <p1> <rating1> <p2> <rating2> - Начало игры
STATE <19 параметров>...     - Состояние игры
DICE <значение>              - Результат броска
RATING_LIST <кол-во>         - Начало таблицы рейтингов
RATING_ENTRY <ранг> <ник> <рейтинг> <победы> <поражения> <ничьи>
RATING_END                   - Конец таблицы рейтингов
GAME_OVER <счет1> <счет2> <победитель> <draw/win/lose>
ERROR <сообщение>            - Ошибка (например, "Column full")

⚠️ Возможные ошибки
1. Login failed!
Неверное имя пользователя или пароль

Пользователь уже авторизован

2. Column full
Выбранный столбец полностью заполнен

Нужно выбрать другой столбец

3. Not your turn!
Попытка бросить кубик не в свой ход

4. Connection issues
Проверьте доступность сервера: ping <IP>

Проверьте порт: telnet <IP> <port>

🎨 Особенности интерфейса
Цветовая индикация:

🟢 Зеленый - ваш ход

🟡 Желтый - ожидание соперника

🔴 Красный - ошибки/проигрыш

🔵 Синий - системные сообщения и справка

Очистка экрана перед каждым обновлением состояния

📊 Система рейтинга
Начальный рейтинг: 1200

Победа: +25 рейтинга

Поражение: -25 рейтинга

Ничья: рейтинг не меняется

🔒 Безопасность
Пароль не отображается при вводе (используется termios)

Все данные передаются в открытом виде (нет шифрования)

🐛 Известные ограничения
Только UDP-соединение (возможна потеря пакетов)

Нет повторного подключения при разрыве

Максимальная длина буфера: 4096 байт

Имена пользователей не должны содержать пробелов

📞 Поддержка
При возникновении проблем проверьте:

Запущен ли сервер

Корректность IP-адреса и порта

Открыт ли порт в брандмауэре

Нет ли конфликта с другими UDP-приложениями
